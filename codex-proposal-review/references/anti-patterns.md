# Anti Patterns

这份文件用来提醒 reviewer：哪些文档“信息很多”，但本质上仍然不可评。

优先在以下场景读取：

- 你感觉文档写得很多，但结论仍然下不去
- 你担心自己被长篇幅、图表或数据量误导
- 你需要向作者解释“为什么这份文档还不能评”

## Anti Pattern 1: 结构完整幻觉

表现：

- 有背景、价值、风险、结论等标准章节
- 但每个章节都停留在表面
- 核心决策问题没有回答

典型误判：

> 结构挺完整的，应该只是细节还要补。

更准确的判断：

> 这不是细节问题，而是核心决策信息仍未到位。

如何识别：

- 读完后仍回答不了“这次到底评什么”
- 读完后仍回答不了“推荐哪个方案、为什么”

## Anti Pattern 2: 数据很多幻觉

表现：

- 有很多表格、截图、交易量、错误码
- 但没有把数据和结论串起来

典型误判：

> 数据量够大，说明分析已经比较充分。

更准确的判断：

> 数据只是材料，关键是这些数据是否支撑了问题定义和方案选择。

如何识别：

- 数据多，但没有结论
- 数据多，但没有影响面判断
- 数据多，但没有把商户影响、渠道差异、方案成本串起来

## Anti Pattern 3: 方案很多幻觉

表现：

- 文档列了 2-4 个备选方案
- 但没有推荐结论
- 也没有决策标准

典型误判：

> 方案已经列得很全了，评审会再讨论一下就行。

更准确的判断：

> 如果推荐方案还没形成，这份文档就还停留在方案收集阶段，不是可拍板阶段。

如何识别：

- 只有“可选方案”，没有“建议方案”
- 没有写清 tradeoff
- 没有写清对商户、运营、渠道的影响

## Anti Pattern 4: 临时止血冒充长期方案

表现：

- 特殊路由
- 原单重发
- 主备切换
- 回切某渠道
- 给重点商户加白

这些可以是必要动作，但不能自动等于“方案已解决”。

典型误判：

> 已经有止血手段，问题算是解决了。

更准确的判断：

> 这说明短期可控，不代表长期能力问题已经解决。

如何识别：

- 文档只写怎么止血
- 没写能力怎么标准化
- 没写未来如何避免重复出现

## Anti Pattern 5: 占位符冒充结论

表现：

- `xx`
- `xxx`
- `待确认`
- `后续补充`
- `评审时讨论`

典型误判：

> 大方向已经有了，这些后面补就行。

更准确的判断：

> 如果占位的是收益、成本、方案归属、商户影响等关键项，这就是未完成决策，不是未完成润色。

如何识别：

- 占位项是否决定是否能通过
- 如果答案是“会”，则应升为 P0

## Anti Pattern 6: comments 已经暴露问题，但正文还在自洽

表现：

- 正文写得像一切都成立
- comments 却反复问“这是什么方案”“为什么会这样”“收益口径是不是错了”

典型误判：

> 正文逻辑还可以，comments 只是个别疑问。

更准确的判断：

> comments 通常比正文更接近真实评审阻塞点。

如何识别：

- 如果 comments 在追问前提，说明正文没有讲清前提
- 如果 comments 在纠正口径，说明正文不是表达不够，而是定义可能有误

## 一条使用规则

当文档命中任意一个 anti-pattern，不要先优化措辞，先判断：

1. 这是不是核心决策问题
2. 这会不会导致评审拍不下来
3. 下一版应该补什么，才能让文档真正进入可评审状态
