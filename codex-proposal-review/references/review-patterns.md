# Review Patterns

这份文件用来快速识别提案属于哪类常见问题模式。

当你读完正文后，先不要急着输出；先判断更接近哪一种模式。

## Pattern 1: 目标混杂

触发信号：

- 同一文档里同时出现“短期机会”“长期建设”“本期先支持”“后续全量开放”
- 评论区反复问“这次到底评审什么”

典型风险：

- 评审结论无法落地
- 会议里所有人都能各讲各的

优先建议：

- 先拆“短期方案”和“长期方案”
- 不要先陷入定价细节

## Pattern 2: 论证断层

触发信号：

- 写了卡点、限制、问题
- 但没说为什么是问题、影响了谁、如何阻碍目标

典型风险：

- 读者知道“有问题”，但不知道“为什么要现在改”

优先建议：

- 每个卡点强制补齐四件事：
- 当前限制
- 形成原因
- 影响目标
- 处理方向

## Pattern 3: 收益口径错误

触发信号：

- comments 明确指出收益不是这个口径
- 文档用 AI 估算、拍脑袋增长率支撑
- 把机会型方案写成普适收入增长故事

典型风险：

- 会把错误的数字补得更漂亮
- 但仍然说服不了评审人

优先建议：

- 先重定义价值来源
- 再决定要不要保留量化收益

## Pattern 4: 定价未决

触发信号：

- 文档列出多个定价方向
- 但没有推荐结论
- 也没说存量商户怎么办

典型风险：

- 方案看似完整，实际关键决策还没做

优先建议：

- 明确推荐一个方向
- 单列写存量商户影响和迁移原则

## Pattern 5: 渠道能力未建模

触发信号：

- 同一支付方式下有多个渠道
- 大家反复提特殊路由、主备、回切、重发
- 不同渠道对字段、目标机构或格式要求不同

典型风险：

- 失败问题永远是“case by case”
- 运营和产品都在临时补洞

优先建议：

- 把问题升级为“能力表达和能力建模”问题
- 不要只写成“某渠道不稳定”

## Pattern 6: 商户影响未评估

触发信号：

- 方案需要改商户入参
- 但没有统计商户数、交易商户数、重点商户

典型风险：

- 方案看似简单，实际落地成本很高

优先建议：

- 补三类数据：
- 签约商户数
- 有交易商户数
- 重点商户/重点客诉商户

## 一条决策规则

如果一份文档同时命中多个模式，优先级通常是：

1. 目标混杂
2. 渠道能力未建模 / 收益口径错误
3. 论证断层
4. 商户影响未评估
5. 定价未决

原因：

- 前两类往往直接决定“这份文档能不能评”
- 后三类更多决定“这份文档评得深不深”
